# 📝 Telegram Chat Logs - Документация

## 🎯 Назначение
Автоматическая отправка истории переписки клиентов с AI-ботом в Telegram группу с созданием отдельных тем для каждой сессии.

## 🔧 Конфигурация

### Переменные окружения (.env)

```env
# Telegram Logs Group Configuration
TELEGRAM_LOGS_BOT_TOKEN=8564716580:AAEag1otwIvGtoGhGAkR31OKNlJaFOvbYBo
TELEGRAM_LOGS_GROUP_ID=-1003743767375
```

**Важно:** Никогда не коммитьте `.env` файл в git! Используйте `.env.example` для шаблона.

## 📋 Структура

### Создание тем
Для каждой новой сессии создаётся отдельная тема в Telegram группе:
- **Формат названия:** `{Имя клиента} - {последние 8 символов session_id}`
- **Пример:** `Клиент - uv2kxqi8n`
- **Цвет иконки:** Синий (0x6495ED)

## 🔁 Кэширование

### Session -> Topic Cache
- **Тип:** In-memory Map
- **Назначение:** Избежание повторного создания тем для одной сессии
- **Ключ:** session_id
- **Значение:** topic_id

```javascript
const sessionTopicCache = new Map();
```

## 📊 Функции

### createTopicForSession(sessionId, userName)
Создаёт новую тему для сессии.

**Параметры:**
- `sessionId` (string) - Уникальный идентификатор сессии
- `userName` (string) - Имя клиента (по умолчанию: 'Клиент')

**Возвращает:** `Promise<number|null>` - topic_id или null

### sendLogToTopic(topicId, message, parseMode)
Отправляет сообщение в указанную тему.

**Параметры:**
- `topicId` (number) - ID темы в Telegram
- `message` (string) - Текст сообщения
- `parseMode` (string) - Режим парсинга ('Markdown' по умолчанию)

**Возвращает:** `Promise<boolean>`

### getOrCreateTopicForSession(sessionId, userName)
Получает ID темы из кэша или создаёт новую.

**Параметры:**
- `sessionId` (string) - Уникальный идентификатор сессии
- `userName` (string) - Имя клиента

**Возвращает:** `Promise<number|null>`

### logChatMessage(sessionId, userName, message, isUser)
Логирует сообщение клиента или бота в Telegram.

**Параметры:**
- `sessionId` (string) - Уникальный идентификатор сессии
- `userName` (string) - Имя клиента
- `message` (string) - Текст сообщения
- `isUser` (boolean) - true если сообщение от клиента, false если от бота

**Возвращает:** `Promise<void>`

## 🎬 Сценарии использования

### Сценарий 1: Новый клиент начинает диалог
1. Клиент пишет первое сообщение
2. Создаётся новая тема в Telegram группе
3. Отправляется приветственное сообщение с информацией о сессии
4. Сообщение клиента логируется в тему
5. Ответ бота логируется в ту же тему

### Сценарий 2: Клиент продолжает диалог
1. Клиент пишет следующее сообщение
2. Тема находится в кэше по session_id
3. Сообщение логируется в существующую тему

### Сценарий 3: Клиент оставил телефон
1. Бот детектирует номер телефона
2. Отправляет лид в Telegram (основной бот)
3. Логирует ответ в тему (logs бот)

### Сценарий 4: Ошибка бота
1. Qwen не отвечает (таймаут)
2. Бот возвращает fallback-ответ
3. Ошибка логируется в тему

## ⚠️ Важные замечания

1. **Бот должен быть администратором группы** с правом создания тем
2. **Группа должна быть супергруппой** (автоматически при создании тем)
3. **Chat ID супергруппы** всегда начинается с `-100`
4. **Кэш in-memory** - при перезапуске сервера кэш сбрасывается
5. **Асинхронное логирование** - ошибки логирования не прерывают работу бота

## 🛠️ Требования

- Telegram Bot API
- Супергруппа с включёнными темами (forum)
- Бот-администратор с правом создания тем
- Node.js 16+
- axios

## 📝 Примеры

### Получение Chat ID группы
```bash
curl -X GET "https://api.telegram.org/bot<YOUR_BOT_TOKEN>/getUpdates"
```

В ответе найдите:
```json
{
  "chat": {
    "id": -1003743767375,
    "title": "Chat Logs",
    "type": "supergroup",
    "is_forum": true
  }
}
```

### Проверка создания темы
```bash
curl -X POST "https://api.telegram.org/bot<YOUR_BOT_TOKEN>/createForumTopic" \
  -H "Content-Type: application/json" \
  -d '{
    "chat_id": -1003743767375,
    "name": "Тестовая тема",
    "icon_color": 0x6495ED
  }'
```
